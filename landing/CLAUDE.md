# Лендинг Meditation — планка качества и плейбук «сделай круто»

Этот файл — контракт уровня для ЭТОГО лендинга (`APP/landing/`, промо на `all-relaxme.ru/promo/`).
Любая сессия, работающая здесь, держит эту планку по умолчанию.

## Что значит «сделай круто» / «ахуенно» / «вау» (главное)

Это НЕ «накидай побольше эффектов». Это:

- **Вау с первого экрана, но со вкусом** — премиально, медитативно, дорого. Уровень портфолио/награды, не «отчёт» и не «AI-дженерик».
- **Передовые техники там, где усиливают смысл** (WebGL/шейдеры, scroll-driven сцены, depth-параллакс, пружинная физика, генеративное движение) — не ради галочки.
- **НЕ перегружать.** Один сильный приём на секцию. Воздух, тишина, ритм паузы. Если рябит/двоится — убрать. «Не перегрузи» — буквальное правило, пользователь его повторяет.
- **Оптимизировано = плавность важнее красоты.** Шёлковый 60fps-скролл, ленивая загрузка, ноль джанка. Лучше меньше эффектов, но идеально гладко.
- **Дизайнерски, на уровне 2026.** Асимметрия, контраст веса шрифта, мягкие свечения через screen-бленд, depth, scroll-нарратив. Никакого бутстрапа/центрованных карточек-плиток.

Планка проверки: «портфолийно, а не как отчёт». Не вау — переделать, а не оправдываться.

## Как работать с пользователем (тот самый коннект)

- Крупная правка → **сначала диалог**: ответь своё мнение/идею («что думаешь?»), накидай план, потом делай. Пользователь ценит соображение, а не молчаливое херачево.
- **Фидбек принимать мгновенно, без защиты.** «Хуйня, верни как было» → вернуть без споров. «Уебански / дёргается / тормозит» → это БАГ, копать причину, а не лепить сверху.
- **Не over-engineer.** Лишние слои, opacity-диссолвы, сущности ради сущностей — нет. Самое простое решение, дающее эффект.
- Авто-режим: не задавать лишних уточнений, делать разумный выбор. Но на развилке дизайна, где можно угадать неверно, — спросить кратко.

## Дизайн-язык

- **Палитра:** фон `#0a0714`, фиолет `#6145c2`, lilac `#d6c8ff`, акценты музыки `#d6c8ff / #9a8cf0 / #e6b8ff`. Тёмно-фиолетовое, медитативное.
- **Шрифты:** display = **Manrope** (Unbounded запрещён как дефолт), mono = JetBrains Mono. Контраст веса: extralight-зачин + medium-акцент. Лесенка/асимметрия как на онбординге аппки.
- **Движение:** framer-motion. Ease по умолчанию `[0.22,0.8,0.36,1]` или `[0.25,1,0.5,1]`. Долгие, мягкие, «дышащие». Реверс на скролле где уместно.
- **Свечения:** через `mix-blend-screen` — чёрных краёв быть не должно. Дым/halo-шейдер — осторожно (тяжёлый; сейчас фон статичный градиент).

## Технический плейбук (уже отлажено — переиспользуй, не изобретай)

- **Scroll-scrub** (`ScrollScrub`): AVIF-секвенция кадров → `canvas.drawImage` по scroll-прогрессу (приём Apple). DPR канвы ≤1.5 (исходники ~1824px), рисуем ТОЛЬКО декодированные кадры (`img._ready` после `decode()` — иначе sync-decode = джанк), пул декода (CONC 6) отложен до `requestIdleCallback`/первого скролла.
- **Depth-шейдер** (`DepthScene`): 1 кадр + карта глубины (Depth Anything), UV-zoom по прогрессу. Виньетка/mouse-параллакс НАРАСТАЮТ с прогрессом (`depthIn`) → стык «последний кадр скраба → depth» бесшовный.
- **Scroll-driven сцены:** `useScroll({target, offset:['start end','center center']})` + `useTransform` → анимация реверсится в обе стороны сама. Скролл сглаживать `useSpring(progress, {stiffness~110, damping~28})`.
- **Огонёк-нарратив:** сквозная искра HowItWorks→Звук, привязана к скроллу (реверс), зажигает свечение на следующей секции. Стыки секций без шва: не клипать свечение `overflow-hidden`, гасить искру на входе/выходе.
- **MouseFloat:** единый слушатель указателя + пружины, лёгкий parallax текстов/контролов по сайту. На тач (`pointer:coarse`) выключен.
- **Аудио-превью** (Звук): нативный HTML5 `Audio`, голос + музыка слоями, `preload="none"`, грузится по клику PLAY.
- **AmorphSphere / шейдеры:** GLSL 1:1 из боевой аппки (`src/shaders`). Screen-бленд. Дорогие — только lazy.

## Жёсткие правила (не нарушать)

- **НЕ запускать headless-браузер / Playwright** на этих WebGL-страницах — вешает Mac пользователя НАГЛУХО (был полный фриз). Проверка = `npm run build` зелёный + глаза пользователя на `localhost:5190`. Визуал не верифицирую headless'ом. (память `feedback_no_headless_screenshots`)
- **Оптимизация загрузки обязательна:** three/r3f и framer — ленивые чанки (`React.lazy` + vite `manualChunks`); всё тяжёлое (WebGL, видео) — `lazy`/по требованию; entry-бандл держать ~200кБ (не давать three вернуться в entry — проверять по выводу build). Кэш-заголовки `immutable` на статику в Caddy.
- **Перф рантайма:** никаких `filter:blur()` по скроллу на крупных элементах (форсит repaint каждый кадр = тормоз) — fade/scale/transform вместо. rAF-циклы паузить за экраном (IntersectionObserver) и в покое. WebGL-контексты минимизировать (`autoPause`/`frameloop:never` вне фазы).
- **Эмодзи в UI — нет**, только SVG-иконки (память `feedback_no_emojis`).
- **Редизайн — только по референсу** от пользователя (память `feedback_no_design_without_reference`).
- **Деплой:** перед — локальный `npm run build`. Затем `git push` → на сервере `git pull && cd landing && npm run build` (SSH по ключу, без пароля). Caddy раздаёт `/promo/` из `/opt/meditation-app/landing/dist`. Полные детали — память `reference_med_app_infra`.

## Чек «достаточно ли круто?» перед тем как сказать «готово»

1. Первый экран цепляет? Премиально, не дёшево?
2. Один сильный приём на секцию — не свалка эффектов?
3. 60fps: скролл / курсор / анимации без джанка? (нет blur-по-скроллу? off-screen rAF паузится? нет лишнего WebGL?)
4. Entry-бандл лёгкий, тяжёлое лениво, кэш есть?
5. Моб выглядит так же вау (и проверено, что не грузит обе версии ассетов)?
6. `npm run build` зелёный.
