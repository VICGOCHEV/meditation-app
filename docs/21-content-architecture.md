## B. Голоса и музыка

### Голос проводника
- **Мужской** — полная дорожка + превью 5–10 сек.
- **Женский** — полная дорожка + превью 5–10 сек.

### Музыка («Энергии»)
Generic-сэмплы, выбираются юзером один раз глобально:
1. **Энергия созидания** — почувствовать лёгкость
2. **Энергия очищения** — почувствовать покой
3. **Энергия жизни** — почувствовать заземление

Превью у каждой — 5–10 сек, для прослушивания при выборе в `VoiceMusicModal`.

Подтверждено клиентом 2026-05-20: это **не привязано к конкретным
практикам** — просто 3 общих сэмпла, чтобы юзер выбрал «энергию», в
которой ему слушать всё.

### Итого по файлам

- Расслабление: 2 + 2 + 6 = **10 mp3**
- Осознанность: 6 × 6 = **36 mp3**
- Авторский: 2 × 6 = **12 mp3** (предположительно)
- Превью: 2 голоса + 3 музыки = **5 mp3**
- **ИТОГО: ~63 mp3** на старт

### Размеры
При 192 kbps и средней длительности 20 мин: ~30 МБ на файл.
**Полный объём ≈ 1.8 ГБ.**

На 20 ГБ SSD (текущий VPS) — поместится. На прод-сервере (если будет
2-4 ГБ Strapi uploads) — тоже ок. Дальше будет нужно S3 / Yandex Object
Storage для масштабирования.

## D. Почему НЕ делаем 2-stream mix

Изначально была идея: голос отдельно, музыка отдельно, миксуем на
клиенте через Web Audio API. Экономия в 6 раз по количеству файлов.

Клиент сказал — **отменяется**: в музыке есть акценты (звук чаш и
т.п.), которые встают чётко в паузы конкретной практики. То есть
музыка не generic-фон, а часть аранжировки. Стемов отдельно не существует.

Поэтому фронт играет **один pre-mixed stream** на практику, выбранный
из 6 вариантов по комбинации `selectedVoice × selectedMusic`.

## E. CMS-схема (нужно обновить)

В Strapi v5 расширить content-type `Practice` с одного `audio`-поля до
**6 nullable audio-полей**:

```yaml
Practice:
  title:                string  required
  block:                enum    relaxation | awareness | author
  description:          text
  price:                number
  order:                int
  month_slot:           int
  duration_sec:         int   (lifecycle хук пересчитывает на upload)

  audio_male_music1:    media
  audio_male_music2:    media   nullable
  audio_male_music3:    media   nullable
  audio_female_music1:  media
  audio_female_music2:  media   nullable
  audio_female_music3:  media   nullable
```

Lifecycle-хук `duration_sec` сейчас читает поле `audio`. Перепишем на
«берём первое непустое из 6 полей, парсим mp3, кладём длительность».
Длительности всех 6 вариантов одинаковые, так что одно значение
покрывает всю практику.

## F. Frontend-логика выбора аудио

В store уже есть `selectedVoice` и `selectedMusic` в `usePlayerStore`.
Логика выбора файла:

```js
const variant = `audio_${voice}_music${music}`           // например audio_male_music3
let url = practice[variant]
// fallback цепочка, если по дизайну переменное число музык
if (!url) url = practice[`audio_${voice}_music1`]        // та же voice, music 1
if (!url) url = practice[`audio_${otherVoice}_music${music}`]
if (!url) url = practice[`audio_${otherVoice}_music1`]
// если всё ещё нет — играть нечего, UX-ошибка
```
