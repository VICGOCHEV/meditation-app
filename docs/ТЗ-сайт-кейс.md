# ТЗ — сайт-кейс «Приложение для медитации и осознанности»

Версия 1.0 · на основе `Кейс-Meditation-App.pdf`

---

## 1. Цель и суть

Сделать **одностраничный сайт-кейс** (portfolio case study), который красиво и
по-человечески рассказывает о проекте «Приложение для медитации». Не презентация
функций, а история: какую задачу решали, что сделали и почему именно так.

Сайт должен:
- произвести впечатление с первого экрана (атмосфера = атмосфера самого приложения);
- провести читателя по понятному нарративу: *что за проект → что сделано → что это дало*;
- закончиться контактом / призывом обсудить похожий проект.

**Главная метрика успеха:** человек долистал до конца и захотел написать.

## 3. Тон и принцип подачи

- Каждый ключевой блок строится по формуле **«сделал X — чтобы Y»** (как в PDF).
- Короткие абзацы, много воздуха, спокойный ритм — как в самом приложении.
- Никаких «реализовано / добавлено / настроено». Только задача → действие → результат.
- Атмосфера важнее плотности: лучше меньше текста, но с дыханием.

## 6. Интерактив и анимации

- **Hero-сфера** — живой WebGL-объект (повтор «капли» из плеера приложения),
  мягко пульсирует. Если тяжело — graceful fallback на CSS-градиентный круг.
- **Дым на фоне** — лёгкий шейдер/нойз за контентом, очень приглушённый.
- **Reveal при скролле** — секции и карточки появляются с fade + лёгкий translateY
  (Framer Motion / Intersection Observer). Без агрессии, спокойно.
- **Плавный скролл** (Lenis) — только на десктопе (pointer: fine), на тачах — нативный.
- **Счётчики/числа** — по желанию count-up при попадании в вьюпорт (перекличка с
  «доезжающими» цифрами замеров в приложении).
- Микро-hover на карточках: лёгкое поднятие границы/свечение акцентом.
- Все анимации — opacity/transform (никаких blur-фильтров на больших областях,
  чтобы не плодить stacking-context и не ронять перформанс).

---

## 7. Адаптив

- **Mobile-first.** Контентная колонка ~`max-width: 720px`, центрирована.
- Брейкпоинты: ≤600 (моб), 600–1024 (планшет), ≥1024 (десктоп).
- Карточки 2×2 на десктопе → 1 колонка на мобиле.
- Hero-заголовок масштабируется (clamp), сфера уменьшается/уходит за текст на узких.
- Строка плашек Утро/День/Вечер/Ночь — горизонтальный ряд → 2×2 на мобиле.

## 8. Технологии

| Слой | Выбор | Почему |
|---|---|---|
| Сборка | Vite + React 18 | Преемственность со стеком проекта; быстрый HMR |
| Стили | Tailwind CSS 3 + токены | Те же токены, что в приложении |
| Анимации | Framer Motion | Reveal, hover, count-up |
| Скролл | Lenis | Плавный скролл на десктопе |
| 3D (опц.) | Three.js + R3F | Hero-сфера; ленивый чанк |
| Хостинг | статика (Vercel / Netlify / Caddy) | Один HTML+JS+CSS бандл |

> Альтернатива «без сборки»: один `index.html` + Tailwind CDN + чистый JS — если
> нужен максимально лёгкий деплой. Решить на старте.

**Важно по производительности:** Three.js и Framer выносим в ленивые чанки,
hero-сфера грузится после первого рендера — первый экран должен открываться
мгновенно (перекличка с оптимизацией лендинга приложения).

## 12. Этапы

1. **Контент-структура** — сверстать секции с текстом, без визуала (каркас читается).
2. **Дизайн-слой** — токены, типографика, карточки, плашки, сэмплы цветов.
3. **Hero + атмосфера** — сфера, дым, градиенты.
4. **Анимации** — reveal при скролле, hover, count-up, Lenis.
5. **Адаптив + производительность** — брейкпоинты, ленивые чанки, оптимизация.
6. **SEO/мета + деплой** — OG, мета, выкладка на хостинг.
