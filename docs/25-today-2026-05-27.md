### 1. IS-баг «всегда шторм»

Все 4 вопроса чекина семантически измеряют ПЛОХОЕ (отвлекают, тревожат,
беспокоят, напряжение) — низкие ответы = хорошее состояние. Старая
формула `calcIS = q1+q2+q3+q4` давала HIGH при плохом состоянии, а
`interpretIS` ждал HIGH = Поток. Полная инверсия логики.

**Фикс**: `calcIS = 40 - (q1+q2+q3+q4)` на фронте + зеркально на бэке.
8/8 unit-тестов проходят. Юзер «4 на всё» теперь → IS=24/40 → Ясность
(вместо Шторма как раньше).

Файлы: `src/utils/scoreCalc.js`, `backend/src/utils/scoreCalc.js`

### 4. TG Mini App auto-auth через initData

- Backend: `verifyTgInitData()` — HMAC-SHA256 через `TG_BOT_TOKEN` по
  стандарту TG (24ч expiry на `auth_date`)
- POST `/api/auth/tg-init {initData}` — upsert User по `tg_user_id`,
  возвращает JWT
- Frontend: `usePlatformAuth` hook ловит `Telegram.WebApp.initData`,
  автоматически логинит. Также — `WebApp.ready()`, `expand()`,
  `setHeaderColor`, `BackButton.show/hide` синхронизирован с location
- Prisma User: `tgUserId BigInt? @unique`, email/passwordHash → optional
- Прод-миграция выполнена через ALTER TABLE

UX: юзер открывает `t.me/Pause_relax_bot/Relaxme` → сразу залогинен,
без формы регистрации.

### 5. VK Mini App auto-auth через vk_signature

- Backend: `verifyVkSign()` — HMAC-SHA256 base64url через `VK_SECURE_KEY`
- POST `/api/auth/vk-init {searchParams}` — upsert User по `vk_user_id`
- Frontend (тот же `usePlatformAuth`): проверяет URL на `vk_user_id+sign`,
  передаёт raw query string на сервер
- Prisma User: `vkUserId BigInt? @unique`
