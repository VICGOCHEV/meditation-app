# 26 — Своя CMS вместо Strapi (спецификация)

Дата: 2026-05-30. Решение клиента: **разочаровались в Strapi, делаем свою**.
Здесь — только дизайн: внешний вид + внутрянка. Код по этой спеке пишется
отдельным заходом.

Подтверждённые на старте решения:

1. **Полная замена Strapi.** Контент переезжает в наш Postgres, фронтовый
   `src/api/cms.js` переключается на наш Fastify-бэк, Strapi гасится.
2. **Охват — всё три:** редактор контента + read-only дашборд (юзеры,
   подписки, чек-ины, КТ) + ручное управление подписками.
3. **Матрица практики 2×3 (голос × музыка) сразу**, с упором на масштаб:
   кнопка «Добавить практику» рождает весь комплект полей автоматически.
   Расчёт на рост 1 → 5 → 10 → 20 практик без боли.

---

## 0. Принцип

Не «ещё одна Strapi», а **узко заточенная панель под ровно наш контент**.
Три сущности, понятная иерархия, ноль абстрактных «collection types».
Каждое поле подписано по-русски, каждое действие — в один-два клика.
Визуально — язык самой аппки (тёмная фиолетовая HUE-тема, Manrope),
**без шейдеров и 3D** — строго плоско-стекло, читаемость прежде всего.

## 1. Контентные единицы (что админим)

### 1.1 Practice (практика) — центральная сущность

| Поле | Тип | Заметка |
|---|---|---|
| `title` | string, required | Название |
| `block` | enum `relaxation / awareness / author` | Блок на главной |
| `description` | text | Короткое описание под названием |
| `price` | int, nullable | Цена поштучной покупки (только `author`) |
| `order` | int | Порядок в блоке (drag-drop) |
| `month_slot` | int, nullable | Слот месячной прогрессии (awareness) |
| `duration_sec` | int, авто | Считается из загруженного mp3 на сервере |
| `published` | bool | Черновик / опубликовано |
| **матрица аудио** | 6 nullable media | `voice(male/female) × music(1/2/3)` |

**Матрица** — главный профит над Strapi. Вместо 6 разрозненных upload-виджетов —
одна сетка 2×3 (см. §3.5). Хранится как 6 nullable FK на медиафайл:

```
audio_male_music1   audio_male_music2   audio_male_music3
audio_female_music1 audio_female_music2 audio_female_music3
```

`music1` (первый столбец) — обязателен для каждого голоса, остальные
nullable: некоторые практики (Утреннее погружение, Обращение к себе) идут
с одной музыкой. Фронт-fallback уже описан в docs/21 §F.

### 1.2 Voice (голос проводника)

| Поле | Тип |
|---|---|
| `name` | string, required (отображаемое) |
| `code` | string, unique, `^[a-z0-9-]+$` (male/female — фронт хранит код) |
| `audio_full` | media, required |
| `audio_preview` | media, required (5–10 сек) |
| `active` | bool |
| `order` | int |

### 1.3 MusicTrack (музыка / «энергия»)

| Поле | Тип |
|---|---|
| `title` | string, required |
| `audio_full` | media, required |
| `audio_preview` | media, required (5–10 сек) |
| `active` | bool |
| `order` | int (фронт мапит порядок → id 1/2/3) |

### 1.4 Read-only / сервисные (из существующего Postgres)

- **Users** — `id, email, name, tgUserId, vkUserId, createdAt`, подписка.
- **Subscriptions** — `tier(awareness/all-inclusive), active, expiresAt`.
  Здесь же — **ручная выдача/продление** (§3.7).
- **Checkins / KtEntry** — последние замеры юзера (только просмотр).

## 2. Внутрянка

### 2.1 Стек — переиспользуем то, что уже стоит

Бэк аппки уже **Fastify + Prisma + PostgreSQL**. CMS = расширение этого же
бэка + отдельный лёгкий React-SPA для админки. Никакого нового рантайма,
никакого Node-демона Strapi на 1+ ГБ RAM.

```
backend/                         (существующий Fastify)
  src/routes/
    admin/
      auth.js          POST /api/admin/login, refresh, logout
      practices.js     CRUD + reorder + публикация
      voices.js        CRUD + reorder
      music.js         CRUD + reorder
      media.js         POST upload (multipart), DELETE
      dashboard.js     GET users / subs / checkins (read-only, пагинация)
      subscriptions.js POST grant / extend / revoke
  prisma/schema.prisma           +Practice +Voice +MusicTrack +MediaFile +AdminUser

admin/                           (новый Vite+React SPA, тот же дизайн-язык)
  собирается в admin/dist → Caddy отдаёт на /admin
```

Контент-эндпоинты для **аппки** (публичные, читают тот же Postgres) — рядом,
повторяют контракт `src/api/cms.js` (см. §2.5), чтобы фронт переключился
сменой одного `VITE_CMS_URL`.

### 2.2 Модель данных (Prisma, новые модели)

```prisma
model MediaFile {
  id        Int      @id @default(autoincrement())
  path      String   // относительный путь в /uploads
  url       String   // публичный URL (/cms-media/...)
  mime      String
  sizeBytes Int
  durationSec Int?   // для аудио — парсим при загрузке
  createdAt DateTime @default(now())
}

model Practice {
  id          Int     @id @default(autoincrement())
  title       String
  block       String  // relaxation | awareness | author
  description String?
  price       Int?
  order       Int     @default(0)
  monthSlot   Int?
  durationSec Int     @default(0)
  published   Boolean @default(false)

  audioMaleMusic1   Int?
  audioMaleMusic2   Int?
  audioMaleMusic3   Int?
  audioFemaleMusic1 Int?
  audioFemaleMusic2 Int?
  audioFemaleMusic3 Int?
  // каждое — relation на MediaFile (6 опциональных связей)

  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
  @@index([block, order])
}

model Voice {
  id           Int     @id @default(autoincrement())
  name         String
  code         String  @unique
  audioFullId  Int
  audioPreviewId Int
  active       Boolean @default(true)
  order        Int     @default(0)
}

model MusicTrack {
  id           Int     @id @default(autoincrement())
  title        String
  audioFullId  Int
  audioPreviewId Int
  active       Boolean @default(true)
  order        Int     @default(0)
}

model AdminUser {
  id           Int    @id @default(autoincrement())
  email        String @unique
  passwordHash String
  role         String @default("editor") // editor | admin
  createdAt    DateTime @default(now())
}
```

> Note: `documentId` Strapi-формата фронт использует как `id` практики
> (`row.documentId || String(row.id)`). При замене — публичный эндпоинт
> отдаёт строковый `id` (можно просто `String(practice.id)`); проверить,
> что отметки прохождения (`PracticeCompletion.practiceId`) не сломаются
> при миграции — см. §2.6.

### 2.3 Хранилище медиа

- Файлы на диск сервера: `/opt/meditation-app/uploads/`.
- Caddy отдаёт их статикой на `/cms-media/*` (тот же домен).
- На загрузке: валидация mime (`audio/mpeg`), лимит размера (~50 МБ),
  серверный парс длительности mp3 (`music-metadata`) → `durationSec`.
- Объём на старте ~1.8 ГБ (docs/21 §C) — влезает. Дальше — Yandex Object
  Storage за тем же интерфейсом `MediaFile.url` (миграция без смены схемы).

### 2.4 Аутентификация

- Отдельная таблица `AdminUser` (НЕ юзеры аппки).
- Логин email+пароль (argon2/bcrypt) → JWT в httpOnly-cookie, короткий
  access + refresh. Все `/api/admin/*` — за `preHandler` проверки роли.
- Стартовый админ создаётся seed-скриптом (креды в `reference_med_app_infra`).
- `subscriptions.js` (выдача подписок) — только `role: admin`.

### 2.5 Публичный контент-API (замена Strapi для аппки)

Повторяем форму ответа, которую сейчас парсит `src/api/cms.js`, чтобы
фронт сменил только базовый URL. Варианты:

- **Проще для фронта:** новый эндпоинт уже отдаёт нормализованную форму
  `{ relaxation:[...], awareness:[...], author:[...] }`, и мы упрощаем
  `cms.js` до тонкого fetch. Практика отдаёт `audioUrl` + новые
  `audio[voice][music]` для матрицы.
- Контракт voices/music — как в `cms.js`: voice → `{id:code,name,fullUrl,previewUrl,active}`,
  music → `{id:1..n,title,fullUrl,previewUrl}`.

Переключение: `VITE_USE_CMS=true` + `VITE_CMS_URL=/api` (наш бэк) вместо `/cms` (Strapi).

### 2.6 Миграция данных из Strapi

Одноразовый скрипт:
1. Тянет из Strapi REST все practices/voices/music + скачивает медиа.
2. Кладёт файлы в `/uploads`, пишет `MediaFile`.
3. Создаёт `Practice/Voice/MusicTrack` с правильными FK.
4. **Сверка `practiceId`:** completions на проде ссылаются на Strapi
   `documentId`. Либо сохраняем старые id как `legacyId` и публичный API
   отдаёт его, либо разово мигрируем `PracticeCompletion.practiceId`.
   → решить на этапе реализации, прод-данные не терять.

### 3.2 Практики — список

Группировка по блокам (это и есть структура главной аппки), внутри блока —
drag-drop сортировка. Карточка-строка, не плотная таблица — дышит.

```
  Практики                                          [ + Добавить практику ]
  ────────────────────────────────────────────────────────────────────────
  РАССЛАБЛЕНИЕ · free                                              3 практики
  ┌──────────────────────────────────────────────────────────────────────┐
  │ ⠿  Утреннее погружение            18 мин   ●●○○○○ 2/6   ● опубл.   ⋯  │
  │ ⠿  Обращение к себе               12 мин   ●●○○○○ 2/6   ● опубл.   ⋯  │
  │ ⠿  Практика расслабления          22 мин   ●●●●●● 6/6   ○ черновик ⋯  │
  └──────────────────────────────────────────────────────────────────────┘
  ОСОЗНАННОСТЬ · по подписке                                       6 практик
  ┌──────────────────────────────────────────────────────────────────────┐
  │ ⠿  Путь к себе                    20 мин   ●●●●●● 6/6   ● опубл.   ⋯  │
  │ ...                                                                    │
  └──────────────────────────────────────────────────────────────────────┘
  АВТОРСКИЙ · поштучно                                              + пусто
```

- `⠿` — хват для drag. `●●○○○○ 2/6` — индикатор заполненности матрицы
  (сколько из 6 дорожек залито) — мгновенно видно, где дыры.
- Бейдж публикации: `●` violet = опубликовано, `○` серый = черновик.
- `⋯` — дублировать / удалить.
- **«Добавить практику»** — главная кнопка (primary-btn градиент).

### 3.3 «Добавить практику» — масштабируемость (ключевое требование)

Клик по кнопке → выезжает редактор **новой пустой практики** со всем
комплектом полей сразу (название, блок, описание, и пустая матрица 2×3 с
6 слотами загрузки). Никаких «сначала создай тип, потом добавь поля» как в
Strapi — комплект полей фиксирован схемой и появляется автоматически.

Рост контента = просто жмёшь «Добавить практику» N раз. Каждая новая строка
самодостаточна. При выборе блока `author` — автоматически показывается поле
`price`; для `awareness` — `month_slot`; для `relaxation` оба скрыты. Поля
«рождаются» по контексту, лишнего на экране нет.

### 3.4 Редактор практики — общий вид

```
  ← Практики / Путь к себе                          [ Сохранить черновик ]  [ Опубликовать ]
  ──────────────────────────────────────────────────────────────────────────────────────
  ┌─ Основное ───────────────────────────┐   ┌─ Параметры ─────────────────┐
  │ Название                              │   │ Блок     [ Осознанность ▾ ] │
  │ [ Путь к себе                      ]  │   │ Порядок  [ 1 ]              │
  │                                       │   │ Слот мес.[ 1 ]   (awareness)│
  │ Описание                              │   │ Длительн. 20 мин  (авто)    │
  │ [                                  ]  │   │ Статус   ○ черновик         │
  │ [                                  ]  │   └─────────────────────────────┘
  └───────────────────────────────────────┘
  ┌─ Аудио-дорожки (голос × музыка) ─────────────────────────────────────────────────────┐
  │                  Музыка 1 ●          Музыка 2 ●          Музыка 3 ●                    │
  │   ♂ Мужской   [▶ male_m1.mp3 ✓]  [▶ male_m2.mp3 ✓]  [  + загрузить  ]                 │
  │   ♀ Женский   [▶ fem_m1.mp3  ✓]  [  + загрузить  ]  [  + загрузить  ]                 │
  └──────────────────────────────────────────────────────────────────────────────────────┘
```

Слева основное (название/описание), справа компактные параметры. Поля
`price`/`month_slot` показываются/прячутся по выбранному блоку (§3.3).
Длительность — read-only, считается сервером из первого залитого файла.

### 3.5 Матрица аудио — звезда панели

Сетка 2 строки (голоса) × 3 столбца (музыки). Каждая ячейка — один из
трёх стейтов:

```
  Пустая:      ┌─────────────────┐    Загрузка:   ┌─────────────────┐
               │   ↑ + загрузить │                │ ▓▓▓▓▓▓░░░ 64%   │
               │   или перетащи  │                │ male_m2.mp3     │
               └─────────────────┘                └─────────────────┘
  Залито:      ┌─────────────────┐
               │ ▶ male_m1.mp3 ✓ │  ← кликабельный мини-плеер (превью прямо в ячейке)
               │ 18:24 · 28 МБ ✕ │  ← длительность, размер, удалить
               └─────────────────┘
```

- **Drag-drop файла прямо в ячейку** или клик «+ загрузить».
- Залитая ячейка — мини-плеер: `▶` слушаешь прямо тут, не скачивая.
- Столбец «Музыка 1» подсвечен как обязательный (тонкая violet-рамка),
  2 и 3 — опциональные (просто `line`-бордер).
- Точка-индикатор у заголовка столбца: есть ли хоть один файл в столбце.
- Это ровно та боль, которую Strapi не умеет: вместо 6 одинаковых
  безымянных media-виджетов — осмысленная сетка, где видно что куда.

### 3.7 Данные — read-only дашборд + подписки

**Юзеры** — таблица с поиском (по email/имени/tg/vk), пагинация. Строка
раскрывается: последние чек-ины, КТ-замеры, статус подписки.

```
  Юзеры                                    🔍 [ поиск по email / tg / vk ]
  ┌──────────────────────────────────────────────────────────────────────┐
  │ ID   Имя / email           Источник   Подписка        Регистрация     │
  │ 142  Анна  anna@..         TG          all-incl ●до 24.06  12.05.2026  │
  │ 141  —     vk:8841         VK          —              11.05.2026       │
  │ ...                                                                    │
  └──────────────────────────────────────────────────────────────────────┘
```

**Подписки** — тот же список, отфильтрован по активным, + действия (только
`role: admin`):

```
  ┌─ Юзер #142 · Анна ───────────────────────────────────────────────────┐
  │ Подписка: all-inclusive · активна до 24.06.2026                       │
  │ [ Продлить +1 мес ]  [ Сменить тариф ▾ ]  [ Отозвать ]                │
  │ ── ручная выдача для теста/саппорта ──                                │
  └──────────────────────────────────────────────────────────────────────┘
```

Подтверждения для деструктива (отозвать/удалить) — модалка. Тарифы и цены —
из `project_med_app_business` (awareness 199 / all-inclusive 299).

### 3.8 Мелочи UX, которые делают «удобно»

- **Автосохранение черновика** + явная кнопка «Опубликовать» (разделение
  draft/published как в Strapi, но без его модели).
- **Inline-валидация:** красная подпись под полем, не алерт.
- **Тосты** в правом нижнем углу на сохранение/ошибку (fade-in анимация
  из аппки, 0.25s).
- **Пустые состояния** с подсказкой и кнопкой («Ещё нет практик в этом
  блоке — Добавить»).
- **Везде мини-плееры** — клиент слышит, что залил, не выходя из CMS.
- **Клавиатура:** `⌘S` сохранить, `Esc` закрыть редактор.
- Полностью **адаптив под планшет** (клиент админит с iPad) — сайдбар
  сворачивается в иконки < 900px.

---

## 4. Что НЕ делаем (scope guard)

- Никаких произвольных content-types / конструктора полей — схема фиксирована.
- Никакой ролевой матрицы кроме `editor`/`admin`.
- Никаких шейдеров, 3D, тяжёлых анимаций — только fade/translate из аппки.
- Никакого rich-text WYSIWYG — описания это простой `<textarea>`.
- Не трогаем логику прогрессии/оплат аппки — CMS только наполняет контент
  и читает данные.
