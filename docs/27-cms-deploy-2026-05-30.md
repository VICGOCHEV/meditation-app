## A. Что построено (локально, в репо)

### Backend (`backend/`) — расширение существующего Fastify
- **Prisma** (`prisma/schema.prisma`): +5 моделей — `MediaFile`, `Practice`
  (матрица 6 audio FK), `Voice`, `MusicTrack`, `AdminUser`.
  SQL миграции: `backend/sql/001_cms_tables.sql` (только
  `CREATE TABLE` — zero-downtime, ничего существующего не трогает).
- **Admin-auth**: `src/middlewares/adminAuth.js` (токены `kind:'admin'`+`aid`,
  отдельный контур от юзеров), `src/routes/admin/auth.js` (login + me).
- **Медиа**: `src/utils/media.js` (стрим на диск + парс длительности mp3),
  `src/routes/admin/media.js` (upload/list/delete).
- **Контент CRUD**: `src/routes/admin/{practices,voices,music}.js`
  (+ reorder, publish-флаг). Сериализация — `src/utils/contentShape.js`.
- **Дашборд + подписки**: `src/routes/admin/dashboard.js` (read-only
  юзеры/чек-ины/КТ + поиск), `src/routes/admin/subscriptions.js`
  (ручная выдача/отзыв, только role `admin`).
- **Публичный контент-API** (замена Strapi для аппки): `src/routes/content.js`
  — `/api/content/{practices,voices,music}`, форма 1-в-1 с `src/api/cms.js`.
- **Seed админа**: `prisma/seed-admin.js`.
- **Миграция со Strapi**: `scripts/migrate-from-strapi.js`.
- Новые env — в `backend/.env.example` (`UPLOAD_DIR`, `MEDIA_URL_BASE`,
  `ADMIN_JWT_TTL`, `MAX_AUDIO_BYTES`).
- Новые npm-зависимости: `@fastify/multipart`, `@fastify/static`,
  `music-metadata`.

### Admin SPA (`cms/`) — отдельный Vite+React в дизайн-языке аппки
- Shell (сайдбар Контент/Данные + топбар), Login.
- Практики: список по блокам с drag-сортировкой, индикатор матрицы `●●○○○○`,
  бейдж публикации; **редактор с матрицей 2×3** (drag-drop файла в ячейку,
  мини-плеер, контекстные поля price/monthSlot).
- Голоса/Музыка: общий `AudioCollection` (full+preview, тумблер active).
- Юзеры: статы, поиск, таблица, drawer с деталями и выдачей подписки.
- Build проверен: `vite build` → ~82 КБ gzip JS.

### Статус проверки
- ✅ Backend стартует, все роуты регистрируются, `/api/health` 200,
  `/api/admin/me` без токена → 401 (auth-гейт работает).
- ✅ SPA собирается без ошибок.
- ⏳ Полный data-e2e (логин → создать практику → залить mp3 → прочитать
  через публичный API) **не гонялся локально** — нет локального Postgres.
  Гонять на проде или на машине с PG.

## B. Деплой на прод (87.228.61.44, /opt/meditation-app)

> **Быстрый путь:** `deploy/deploy-cms.sh` делает шаги 1, 3, 4 автоматически
> (идемпотентно). Caddy-блоки готовы в `deploy/cms.Caddyfile`. Ниже —
> те же шаги вручную + ручные шаги (seed/миграция/cutover), которые скрипт
> намеренно не делает.

### 1. Код + зависимости + Prisma
```bash
ssh root@87.228.61.44
cd /opt/meditation-app
git pull --ff-only

# backend
cd backend
npm install
# применить новые таблицы (zero-downtime, только CREATE TABLE):
psql "$DATABASE_URL" -f sql/001_cms_tables.sql   # путь от backend/
npx prisma generate

# создать папку загрузок (вне репо, чтобы git pull не сносил)
mkdir -p /opt/meditation-app/uploads
chown -R <api-user>: /opt/meditation-app/uploads
```

Добавить в `backend/.env`:
```
UPLOAD_DIR=/opt/meditation-app/uploads
MEDIA_URL_BASE=/cms-media
ADMIN_JWT_TTL=12h
MAX_AUDIO_BYTES=62914560
```

Перезапуск API:
```bash
systemctl restart meditation-api
curl -s localhost:3001/api/health   # ждём {"ok":true}
```

### 2. Создать первого админа
```bash
cd /opt/meditation-app/backend
ADMIN_EMAIL=client@all-relaxme.ru ADMIN_PASSWORD='СильныйПароль' ADMIN_ROLE=admin \
  node prisma/seed-admin.js
# креды записать в reference_med_app_infra (память)
```

### 3. Собрать admin SPA
```bash
cd /opt/meditation-app/cms
npm install
npm run build       # → cms/dist
```

### 4. Caddy — отдать /admin и /cms-media
В Caddyfile сайта `all-relaxme.ru` (порядок важен: специфичные пути выше
общего fallback фронта):

```caddyfile
all-relaxme.ru {
    # ... существующее: /api → :3001, статика фронта ...

    # CMS-админка (SPA из папки cms/, путь /admin/ чтобы не конфликтовать
    # со Strapi на /cms/ во время перехода; отдаём index.html на все пути)
    handle_path /admin/* {
        root * /opt/meditation-app/cms/dist
        try_files {path} /index.html
        file_server
    }

    # Загруженное аудио CMS
    handle_path /cms-media/* {
        root * /opt/meditation-app/uploads
        file_server
    }

    # ... /api проксируется к :3001 (там же теперь /api/admin/* и /api/content/*) ...
}
```
```bash
caddy validate --config /etc/caddy/Caddyfile && systemctl reload caddy
```
Проверка: открыть `https://all-relaxme.ru/admin/` → экран логина.

### 5. Перенос контента со Strapi
```bash
cd /opt/meditation-app/backend
STRAPI_BASE=https://all-relaxme.ru/cms node scripts/migrate-from-strapi.js
```
Скрипт скачает аудио в `uploads/`, создаст MediaFile + Practice/Voice/Music.
Практики матчатся по `slug = Strapi documentId` → **прогресс юзеров
(`PracticeCompletion.practiceId`) сохраняется**. Проверить в `/admin`, что
всё на месте, послушать дорожки.

### 6. Переключить аппку на свой контент-API
Когда контент проверен в `/admin`:
- Фронт аппки: `VITE_CMS_URL=/api/content` (вместо `/cms`), пересобрать,
  выкатить. `src/api/cms.js` уже отдаёт совместимую форму — менять логику
  не нужно, только базовый URL (см. §C ниже про нюанс путей).
- Smoke-тест на проде: главная → выбор голоса/музыки → плеер играет →
  завершение → трекер отметился.

### 7. Погасить Strapi (после успешного smoke-теста)
```bash
systemctl stop meditation-cms && systemctl disable meditation-cms
# Caddy: убрать /cms reverse-proxy. БД/файлы Strapi не удалять ~2 недели.
```

## C. Нюанс: путь публичного API

`src/api/cms.js` строит запросы как `${BASE}/api/practices` и медиа как
`${BASE}${media.url}`. Наш публичный API — `/api/content/practices`, а медиа
уже отдаёт абсолютный `/cms-media/...`. Два варианта:

1. **Тонкая правка `cms.js`** (рекомендуется): новые функции читают
   `/api/content/*`, медиа-URL берут как есть (уже готовый). Это финальный
   cutover-PR, делается на шаге 6 — пока Strapi жив, не трогаем.
2. Или повесить на Caddy alias `/cms/api/* → /api/content/*` для полной
   прозрачности. Грязнее, но без правки фронта.

Реализуем вариант 1 при cutover.

## D. Откат

- **SPA/Caddy**: убрать `handle_path /admin` и `/cms-media`, reload caddy.
- **API**: новые роуты аддитивны и за auth — не мешают аппке; можно просто
  не переключать фронт (он останется на Strapi `/cms`).
- **БД**: новые таблицы не трогают старые. При полном откате —
  `DROP TABLE "MediaFile","Practice","Voice","MusicTrack","AdminUser" CASCADE;`
- Strapi не гасим до подтверждённого smoke-теста — мгновенный откат фронта
  на `/cms`.
