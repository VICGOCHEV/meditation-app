## C. Можно делать параллельно (не блокируется клиентом)

### C1. Оплата — заглушка ЮKassa
Подготовить серверную интеграцию с тест-ключами ЮKassa (создание заказа, webhook, активация подписки), даже если боевых ключей ещё нет — сразу проверим на их sandbox.

### C2. SMS-флоу — каркас
Реализовать `/api/auth/verify` как заглушку, которая принимает любой `code === "0000"` в dev-режиме. Когда клиент пришлёт провайдера — подменить только адаптер.

### C3. Push (Web Push, VAPID)
Сгенерировать VAPID-ключи, добавить `/api/push/subscribe`, написать тест-эндпоинт для отправки. Тексты подставим, когда придут.

### C4. Удаление аккаунта (GDPR / Apple/Google)
`DELETE /api/auth/me` с каскадом по всем таблицам пользователя + кнопка в Profile. Требование платформ; можем сделать до того, как спросят.

### C5. S3-хранилище для аудио
Сейчас Strapi отдаёт mp3 с локального диска VPS. Для боевой нагрузки — провайдер S3-совместимого storage (Yandex Object Storage). Strapi v5 это умеет через `@strapi/provider-upload-aws-s3`. Подключается за вечер, когда клиент скажет, какой бакет.

### C6. CI/CD
Сейчас деплой = ручной rsync. Минимум — git-хук на VPS, который дёргает `npm run build` после `git pull`. По-хорошему — GitHub Actions → SSH-деплой.

### C7. Бэкапы Postgres
Cron-таск с `pg_dump` + ротация. День работы.

### C8. Десктоп-вёрстка (F6)
Сейчас всё в `max-w-md`. Решить с клиентом: оставить мобильную колонку по центру (как TG Web) или сделать полноценный десктоп. Можем подготовить оба прототипа.
